Visualizing Neural Networks

A practical map of the six core components that make a neural network learn
Derrick Karake · AIML-501 Workshop 2 · Portfolio Artifact

Architecture — How Data Flows

Input Layer (raw features) Hidden Layers (learned representations) Output Layer (prediction)
1

Layers

Stacks of neurons grouped by role.
A neural network is organized into an input layer (raw features), one or more hidden layers (learned representations), and an output layer (prediction). Each layer refines what the previous one passed along, transforming inputs into successively more abstract features.
Example: a 3-layer image classifier might use the input layer for pixel values, hidden layers for edges and shapes, and the output layer for class probabilities.
2

Neurons

Single units that compute a weighted sum and pass it through a non-linearity.
Each neuron takes inputs, multiplies them by its weights, adds a bias, and feeds the result through an activation function. The output becomes input for the next layer. Many simple neurons working together approximate very complex functions.
Example: a neuron in a sentiment model might fire strongly when its inputs combine to indicate negative tone.
3

Weights

The learnable parameters that decide how much each input matters.
Weights are the dials the network turns during training. A larger weight on an input means that input has a stronger influence on a neuron's output. The entire "knowledge" of a trained network lives in its weight values.
Example: in a fraud detector, the weight on "transaction amount" usually grows larger than the weight on "transaction time" because the size matters more than the hour.
4

Activation Functions

Non-linear transformations applied to each neuron's output.
Without activations, stacking layers is just stacking linear functions — no extra power. Activations like ReLU, sigmoid, and tanh introduce non-linearity so the network can model curved, complex relationships. ReLU is the modern default for hidden layers.
Example: ReLU passes any positive value through unchanged and zeros out negatives — simple, fast, and surprisingly effective.
5

Loss Functions

A single number that scores how wrong the network's prediction was.
A loss function compares the network's output to the correct answer and produces an error score. Training is the process of making this number as small as possible. Different problems use different losses: cross-entropy for classification, mean squared error for regression.
Example: if a model predicts 0.7 probability of "dog" but the true label is 1.0, cross-entropy gives a positive loss that the optimizer will then try to shrink.
6

Optimization Algorithms

The recipe for adjusting weights to reduce the loss.
Optimizers use backpropagation to compute how each weight contributed to the error, then nudge each weight in the direction that lowers the loss. Stochastic Gradient Descent (SGD), Adam, and RMSProp are the most common choices. Adam is the modern default because it adapts the step size per parameter.
Example: after each mini-batch, Adam updates every weight a small amount — over many batches the weights converge to values that produce accurate predictions.

Why Visualizing Neural Networks Matters

Translating equations into diagrams changed how I think about neural networks. The math always describes the same six pieces — layers, neurons, weights, activations, loss, and optimizers — but the picture makes their relationships immediate. You can see why a deeper network learns richer features, why activations matter, and why the loss function is what shapes everything that happens in training.

For non-technical stakeholders, this kind of artifact is the difference between "trust me, the model works" and "here is how the model works." That second version is the one that opens doors in industry — it lets engineers, product managers, and business leaders speak the same language about what an AI system is actually doing.

Going forward, I plan to keep updating this artifact each time I learn a new architecture family — CNNs, RNNs, Transformers — so the visual library grows alongside my technical depth.
